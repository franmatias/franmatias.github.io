SMACSS

Scalable and Modular Architecture for CSS

¿Qué es SMACSS?

SMACSS (Scalable and Modular Architecture for CSS) es una metodología de diseño creada por Jonathan Snook que proporciona una forma estructurada de organizar las reglas CSS de un proyecto. No es un framework, sino más bien un estilo de guía que ayuda a:

  • Aumentar la semántica del código
  • Incrementar la reutilización de código
  • Mejorar la mantenibilidad del proyecto
  • Reducir la especificidad de los selectores

Categorías SMACSS

1. Base

Los estilos base son los estilos predeterminados para elementos HTML. No incluyen clases ni IDs, solo selectores de elementos, selectores de atributos, pseudo-clases, selectores hijos o hermanos.

body {
    font-family: Arial, sans-serif;
}

a {
    color: #0066cc;
}

input[type="text"] {
    border: 1px solid #ccc;
}

2. Layout

Los estilos de layout dividen la página en secciones. Layouts sostienen uno o más módulos juntos. Se prefijan con "l-" o "layout-".

.l-header {
    width: 100%;
    position: fixed;
}

.l-sidebar {
    float: left;
    width: 200px;
}

3. Módulo

Los módulos son los componentes reutilizables y modulares de nuestro diseño. Son la mayor parte de su sitio.

.card {
    background: #fff;
    border-radius: 4px;
}

.card__title {
    font-size: 1.2em;
}

.card__content {
    padding: 1em;
}

4. Estado

Los estilos de estado describen cómo nuestros módulos o layouts lucirán en un estado particular. Se prefijan con "is-" o "has-".

.is-hidden {
    display: none;
}

.is-active {
    background: #f0f0f0;
}

.has-error {
    border-color: red;
}

5. Tema

Los estilos de tema son similares a los estilos de estado pero definen colores y aspectos visuales que dan un aspecto específico a módulos o layouts.

.theme-dark {
    background: #333;
    color: #fff;
}

.theme-holiday {
    background: url('snowflakes.png');
    color: #red;
}

Mejores Prácticas SMACSS

  • Minimizar la profundidad de los selectores: Usar clases específicas en lugar de selectores anidados.
  • Evitar IDs: Los IDs tienen una especificidad muy alta y son menos reutilizables.
  • Usar prefijos significativos: Como l- para layouts, is- para estados, theme- para temas.
  • Organizar CSS por categorías: Mantener archivos separados para cada categoría SMACSS.
  • Mantener la modularidad: Crear módulos independientes que puedan reutilizarse.
  • Documentar el código: Incluir comentarios que expliquen la función de cada módulo.
